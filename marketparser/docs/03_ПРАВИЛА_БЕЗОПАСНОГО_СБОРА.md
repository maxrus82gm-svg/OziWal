# 03. Правила безопасного сбора данных

> Навигация: [[marketparser/Карта_анализа_рынка|Карта анализа рынка]]

Этот документ описывает безопасные правила будущего сбора данных для `marketparser`.

## Главный принцип

Парсер нужен для исследовательской оценки идей, а не для агрессивного копирования маркетплейса.

## Правила безопасного сбора данных

1. На первом этапе использовать ручную или полуавтоматическую проверку.
2. Не делать массовые запросы к сайтам.
3. Не обходить капчи.
4. Не обходить авторизацию.
5. Не ломать антибот-защиту.
6. Не использовать скрытые или сомнительные методы доступа.
7. Не собирать персональные данные покупателей.
8. Не собирать данные, не относящиеся к анализу товарных идей.
9. Не хранить токены, cookies, логины и пароли в репозитории.
10. Если в будущем используется API, ключи должны храниться вне Git.
11. Любой автоматический сбор данных должен быть ограничен по объёму.
12. Для старта достаточно 10-20 запросов и топ 10-20 товаров по каждому запросу.

## Что можно считать достаточным для старта

Для первой исследовательской проверки достаточно вручную или полуавтоматически собрать:

- поисковый запрос;
- площадку;
- диапазон цен;
- количество отзывов у похожих товаров;
- примерный рейтинг;
- уровень конкуренции;
- возможность сделать товар лучше;
- сложность моделирования и печати.

## Что запрещено без отдельной задачи

- подключаться к OZON или Wildberries;
- создавать API-клиент;
- создавать рабочий парсер;
- создавать скрипты автоматического обхода страниц;
- сохранять токены, cookies, логины или пароли;
- создавать массовые выгрузки.
