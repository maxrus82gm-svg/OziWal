# 02. Структура данных

> Навигация: [[marketparser/Карта_анализа_рынка|Карта анализа рынка]] · Назад: [[marketparser/docs/01_ПРОЦЕСС_ИССЛЕДОВАНИЯ_СПРОСА|Процесс исследования спроса]] · Далее: [[marketparser/docs/03_ПРАВИЛА_БЕЗОПАСНОГО_СБОРА|Правила безопасного сбора]]

Этот документ описывает текущую и планируемую структуру данных `marketparser`.

Каталоги `data/input`, `data/manual`, `docs` и `reports` уже используются. Каталоги для автоматического сбора и код создаются только по отдельной задаче.

## Текущая и планируемая структура

```text
marketparser/
├── data/
│   ├── input/
│   │   └── product_ideas.csv
│   ├── manual/
│   │   └── manual_market_check.csv
│   ├── raw/
│   │   ├── ozon_raw.csv
│   │   └── wb_raw.csv
│   └── processed/
│       └── product_ideas_score.csv
├── reports/
│   └── demand_report.md
└── src/
    └── будущий код парсера
```

## Назначение папок

- `data/input` — исходные идеи и запросы.
- `data/manual` — данные ручной проверки.
- `data/raw` — сырые выгрузки будущего парсера.
- `data/processed` — обработанные оценки.
- `reports` — отчёты.
- `src` — будущий код, только после отдельной задачи.

## Правило текущего этапа

На текущем этапе без отдельной задачи не создавать:

- `marketparser/data/raw`;
- `marketparser/data/processed`;
- `marketparser/src`;
- массовые выгрузки;
- рабочий код.
