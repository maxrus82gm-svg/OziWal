# Manual Market Check Guide

## Назначение

Этот документ описывает, как вручную проверять спрос по запросам из `marketparser/data/manual/manual_market_check.csv`.

Шаблон нужен для аккуратного внесения результатов ручной проверки. Он не является парсером и не содержит результатов автоматического сбора данных.

## Что проверяем

Для каждого запроса нужно вручную посмотреть ограниченное количество видимых товаров на OZON и/или Wildberries.

Проверка нужна для предварительной оценки:

- есть ли выдача;
- есть ли похожие товары;
- какие примерно цены;
- есть ли отзывы;
- насколько сильная конкуренция;
- можно ли сделать товар лучше;
- есть ли IP-риск.

## Ограничения

- Не делать массовый парсинг.
- Не обходить капчи.
- Не обходить авторизацию.
- Не использовать cookies, логины, пароли или токены.
- Не сохранять HTML-страницы маркетплейсов.
- Не скачивать изображения товаров.
- Не копировать чужие описания и названия как готовые карточки.
- Не проверять слишком много запросов за один раз.
- Для первого теста достаточно 5-10 запросов.

## Как заполнять поля

- `check_id` — внутренний ID проверки, например `CHECK-001`.
- `idea_id` — ID идеи из `marketparser/data/input/product_ideas.csv`.
- `query` — поисковый запрос для ручной проверки.
- `marketplace` — площадка проверки: `OZON`, `WB` или `OZON_WB`.
- `access_status` — доступность ручной проверки: `not_checked`, `available`, `unavailable`, `captcha`, `error`.
- `results_visible` — видна ли выдача: `unknown`, `yes`, `no`.
- `top_items_checked` — сколько карточек просмотрено вручную.
- `approx_price_min` — примерная минимальная цена среди видимых похожих товаров.
- `approx_price_max` — примерная максимальная цена среди видимых похожих товаров.
- `reviews_max_visible` — максимальное видимое количество отзывов среди похожих товаров.
- `rating_visible` — виден ли рейтинг: `unknown`, `yes`, `no`.
- `competition_level` — примерная конкуренция: `unknown`, `low`, `medium`, `high`.
- `photo_quality_level` — уровень фото у конкурентов: `unknown`, `low`, `medium`, `high`.
- `same_products_many` — много ли одинаковых товаров: `unknown`, `yes`, `no`.
- `can_make_better` — можно ли сделать лучше: `unknown`, `yes`, `no`, `maybe`.
- `ip_risk` — риск по брендам и IP: `low`, `medium`, `high`.
- `safe_alternative` — безопасная альтернатива для рискованных запросов.
- `manual_status` — статус ручной проверки: `not_checked`, `checked`, `needs_recheck`.
- `notes` — короткий комментарий по проверке.

## Правила по IP-рискам

- Игровые бренды и названия игр можно использовать только как поисковые гипотезы.
- Нельзя считать брендовые запросы безопасными производственными идеями.
- Для high-risk запросов обязательно указывать безопасную альтернативу.
- Производственная идея должна опираться на оригинальную безопасную формулировку.
- Нельзя копировать персонажей, логотипы, названия фракций, эмблемы и узнаваемые предметы.

## Как оценивать конкуренцию

- `low` — мало похожих товаров, слабая выдача.
- `medium` — есть похожие товары, но можно выделиться дизайном.
- `high` — много похожих товаров, сильные карточки, много отзывов, низкие цены.

## Как оценивать возможность сделать лучше

Оценивать по признакам:

- можно ли сделать более красивую форму;
- можно ли сделать персонализацию;
- можно ли сделать набор;
- можно ли улучшить фото;
- можно ли сделать уникальный стиль;
- можно ли избежать IP-рисков.

## Следующий шаг

После заполнения первых строк `manual_market_check.csv` можно будет сделать следующую задачу:

проанализировать ручные результаты и выбрать 3-5 идей для более глубокой проверки или тестовой печати.
