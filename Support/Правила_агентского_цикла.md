# Правила агентского цикла проекта

## Назначение документа

Этот документ является постоянным источником правил агентского цикла проекта.

Он описывает:

- назначение;
- жизненный цикл TASK;
- роли участников;
- правила выполнения;
- правила приёмки;
- правила внесения изменений.

---

## Источник истины (Single Source of Truth)

Каждый тип информации хранится только в одном месте.

### Правила Single Source of Truth:

- **Рабочая среда** → `Support/Рабочая_среда.md`
- **Агентский цикл** → `Support/Правила_агентского_цикла.md`
- **Текущая задача** → `БЛОК 1` (в `Support/00_BIG_TASK_CONTEXT.md`)
- **Последний CODEX REPORT** → `БЛОК 2` (в `Support/00_BIG_TASK_CONTEXT.md`)
- **Последний REVIEW** → `БЛОК 3` (в `Support/00_BIG_TASK_CONTEXT.md`)
- **Постоянные правила проекта** → отдельные документы в `Support/`

**Дублирование постоянных правил между документами запрещено.**

---

## Роли участников

### Пользователь

- определяет направление проекта;
- принимает архитектурные решения;
- утверждает изменения процесса;
- выполняет Commit;
- выполняет Push.

### Ассистент

- является архитектором проекта;
- формирует TASK;
- определяет архитектурные решения;
- выполняет REVIEW;
- принимает либо отклоняет результат;
- не выполняет реализацию вместо агента.

### Codex

- является исполнителем;
- выполняет TASK;
- соблюдает правила проекта;
- публикует итоговый отчёт;
- не выполняет Commit;
- не выполняет Push.

---

## Правило Report Stop

После публикации TASK начинается этап выполнения.

До получения отчёта агента:

- **БЛОК 1 считается замороженным**;
- требования TASK изменять запрещено;
- критерии выполнения изменять запрещено;
- новые требования в БЛОК 1 добавлять запрещено.

Все возникающие уточнения публикуются только в чате.

**Получение отчёта агента является контрольной точкой (Report Stop).**

После получения отчёта ассистент выполняет REVIEW.

---

## Неизменяемость БЛОКА 1

После публикации TASK запрещается:

- изменять требования;
- удалять требования;
- добавлять требования;
- изменять критерии приёмки.

Если во время выполнения появляются новые идеи, замечания или дополнительные требования, они оформляются новой TASK после завершения текущего цикла.

---

## Обязательный отчёт агента

После завершения каждой TASK агент обязан опубликовать краткий отчёт в чат.

### Минимальная структура отчёта:

```
━━━━━━━━━━━━━━━━━━━━━━

TASK XXX — Выполнено

Изменено файлов: N

Изменённые файлы:

• ...
• ...

Кратко выполнено:

• ...
• ...

Commit: нет

Push: нет

Ожидаю REVIEW ассистента.

━━━━━━━━━━━━━━━━━━━━━━
```

Допускается расширение отчёта дополнительной полезной информацией.

---

## REVIEW

Работа считается завершённой только после REVIEW ассистента.

### Статусы процесса:

- TASK опубликована;
- TASK выполняется;
- Report Stop;
- REVIEW;
- Принято.

**или**

- Новая TASK.

**Выполнение задачи не означает её автоматическую приёмку.**

---

## Исправления после REVIEW

Если после REVIEW обнаружены замечания:

- опубликованная TASK не изменяется;
- замечания не добавляются в БЛОК 1;
- создаётся новая TASK.

---

## Формат публикации

Любой текст, предназначенный для полного копирования пользователем (TASK, инструкции, промпты, конфигурации, шаблоны и т.п.), публикуется **одним непрерывным блоком**.

**Запрещается разбивать такой текст на несколько отдельных блоков.**

Все TASK должны публиковаться единым блоком, готовым для копирования одной кнопкой «Копировать» и последующей вставки без дополнительного редактирования.

**Исключение** допускается только по явной просьбе пользователя.

---

## Жизненный цикл TASK

1. **Формирование** — ассистент создаёт TASK.
2. **Публикация** — TASK помещается в БЛОК 1.
3. **Выполнение** — агент выполняет задачу.
4. **Report Stop** — агент публикует отчёт.
5. **REVIEW** — ассистент проверяет результат.
6. **Приёмка** — ассистент утверждает результат.
7. **Завершение** — БЛОК 1 очищается, статус записывается в БЛОК 3.

## Обновление документа

Документ может обновляться только при принятии архитектурного решения пользователем и утверждении ассистентом.

Изменения фиксируются новой версией документа с описанием изменений.
