# Правила агентского цикла проекта

> Навигация: [[Support/Карта_документации|Карта документации]]

## Назначение

Этот документ является единым источником истины для агентского цикла проекта.

Только здесь подробно определяются:

- роли Пользователя, Ассистента и Агента;
- режимы выполнения;
- полный запрет Git для Агента;
- жизненный цикл TASK;
- контрольная точка отчёта;
- единый краткий формат ОТЧЁТА АГЕНТА;
- порядок Commit и Push;
- проверка нового Commit и REVIEW Ассистентом;
- решения после REVIEW, исправления и откат.

## Единый источник истины

Каждый тип информации хранится в одном месте:

- агентский цикл, Git-политика, отчёт и REVIEW → этот документ;
- публикация TASK и обновление БЛОКОВ 1–3 → `Support/Правила_запуска_задач.md`;
- локальная работа с файлами, резервные копии, Windows, PowerShell и UTF-8 → `Support/02_ПРАВИЛА_ЛОКАЛЬНОЙ_РАБОТЫ_АГЕНТА.md`;
- рабочие места, пути и выбранные режимы → `Support/Рабочая_среда.md`;
- активная TASK, последний отчёт и последний статус → `Support/00_BIG_TASK_CONTEXT.md`;
- правила карточек товаров → `Support/01_ПРАВИЛА_КАРТОЧЕК_ТОВАРОВ.md`;
- правила Excel → `Support/06_ПРАВИЛА_БЕЗОПАСНОЙ_РАБОТЫ_С_EXCEL.md`;
- правила анализа рынка → `marketparser/Карта_анализа_рынка.md`.

Подробное дублирование этих правил в других документах запрещено. Другие документы дают только краткую ссылку на соответствующий источник.

## Роли

### Пользователь

- определяет направление проекта и утверждает архитектурные решения;
- публикует TASK и короткое сообщение запуска;
- выполняет все операции Git, включая Commit, Push и откат;
- принимает решение о публикации изменений.

### Ассистент

- проверяет актуальное состояние `main` через GitHub;
- показывает стартовый и итоговый статус Commit;
- формирует TASK, ограничения и критерии готовности;
- после Push изучает новый Commit, diff и фактические файлы;
- выполняет REVIEW;
- рекомендует принятие, новую TASK, частичный или полный откат.

### Агент-исполнитель

Агент-исполнитель — локальная ИИ-система, которая выполняет активную TASK в разрешённых границах. Конкретным Агентом может быть Codex или другая агентская система.

Агент:

- работает только с локальными файлами;
- соблюдает режим и границы TASK;
- изменяет только разрешённые файлы;
- выполняет разрешённые локальные проверки;
- публикует краткий ОТЧЁТ АГЕНТА;
- не принимает окончательное решение о приёмке TASK.

## Режимы выполнения

### Ограниченный

- Ассистент заранее определяет реализацию, файлы и точные изменения;
- Агент выполняет только прямо указанные действия;
- Агент не выбирает альтернативную реализацию и не добавляет улучшения;
- при неоднозначности Агент останавливается и сообщает о проблеме.

### Самостоятельный

- Ассистент определяет цель, архитектуру, ограничения и критерии готовности;
- Агент самостоятельно исследует связанные локальные файлы;
- Агент выбирает безопасный локальный способ реализации;
- Агент выполняет необходимые локальные проверки;
- Агент исправляет только ошибки, непосредственно связанные с TASK;
- Агент не меняет общую архитектуру и не выходит за границы TASK.

### Выбор режима

- режим, явно указанный в TASK, имеет высший приоритет;
- если режим в TASK не указан, используется режим текущего рабочего места из `Support/Рабочая_среда.md`;
- Агент не может самостоятельно менять режим.

## Полный запрет Git для Агента

Все операции Git выполняет только Пользователь.

Агенту без исключений запрещено:

- выполнять любые Git-команды;
- выполнять `git status`, `git diff`, `git log` и `git add`;
- выполнять Commit, Push, Pull и Fetch;
- создавать или переключать ветки;
- изменять Git-конфигурацию;
- выполнять частичный или полный Git-откат;
- обращаться к GitHub или другим удалённым репозиториям;
- использовать сеть для проверки репозитория.

Прямое указание Пользователя внутри обычной TASK не отменяет этот запрет.

Изменить запрет можно только отдельным архитектурным решением Пользователя и новой TASK по изменению постоянных правил.

Краткая обязательная формулировка для других документов:

> Любые Git-команды запрещены согласно `Support/Правила_агентского_цикла.md`.

## Неизменяемость опубликованной TASK

После публикации TASK и до ОТЧЁТА АГЕНТА БЛОК 1 заморожен.

Запрещено задним числом:

- менять или удалять требования;
- добавлять требования;
- менять критерии готовности;
- подменять выполненную TASK уточнённой версией.

Уточнения передаются в чате. Замечания после выполнения оформляются новой TASK.

## Жизненный цикл TASK

1. Ассистент проверяет актуальную вершину `main` и показывает СТАРТОВЫЙ КОММИТ.
2. Ассистент формирует полную TASK и короткое сообщение запуска.
3. Пользователь публикует TASK в БЛОКЕ 1.
4. Пользователь отправляет короткое сообщение Агенту.
5. Агент выполняет TASK локально без Git и сети.
6. Агент публикует краткий ОТЧЁТ АГЕНТА.
7. При полном выполнении Агент записывает отчёт в БЛОК 2, предварительный статус в БЛОК 3 и очищает содержимое БЛОКА 1 с сохранением оболочки.
8. При частичном выполнении или ошибке Агент не очищает БЛОК 1 полностью.
9. Пользователь выполняет Commit и Push.
10. После получения отчёта Ассистент сразу запрашивает актуальную вершину `main` через GitHub.
11. Ассистент показывает СТАТУС КОММИТА.
12. Ассистент без дополнительного запроса открывает новый Commit, изучает diff, читает изменённые документы, проверяет созданные, удалённые, переименованные и перемещённые файлы, сверяет результат с TASK и выполняет REVIEW.
13. Ассистент принимает одно решение: принять результат, создать новую TASK на исправление, рекомендовать частичный откат или рекомендовать полный откат.
14. Завершённая TASK не переписывается задним числом.
15. Все замечания оформляются новой TASK.

## Контрольная точка отчёта

ОТЧЁТ АГЕНТА завершает локальный этап выполнения, но не означает принятие результата.

После получения отчёта Ассистент:

1. кратко читает отчёт;
2. отмечает только очевидные критические нарушения;
3. сразу проверяет актуальную вершину `main`; 
4. не проводит длинную предварительную проверку только по тексту отчёта.

После Push ОТЧЁТ АГЕНТА не является источником истины. Источники истины:

- новый Commit;
- diff;
- фактическое содержимое репозитория GitHub.

Проверка старого Commit вместо нового результата запрещена.

## Статус нового Commit

Если новый Commit обнаружен, Ассистент выводит:

    СТАТУС КОММИТА

    Repository: owner/repository
    Branch/ref: main
    Commit SHA: полный SHA
    Commit message: точное сообщение
    Проверка: выполнена через GitHub

После этого Ассистент сразу выполняет фактический REVIEW.

Если новый Commit не обнаружен, Ассистент выводит:

    НОВЫЙ КОММИТ НЕ ОБНАРУЖЕН

    Repository: owner/repository
    Branch/ref: main
    Последний известный Commit SHA: полный SHA
    Последнее сообщение Commit: точное сообщение

После этого Ассистент ожидает Commit и Push Пользователя.

## Решения после REVIEW

### Принять

Результат соответствует TASK, критических ошибок нет.

### Новая TASK на исправление

Основной результат полезен, откат не нужен, но необходимы отдельные исправления.

### Частичный откат

Нужно отменить только отдельные ошибочные изменения.

### Полный откат

Результат в целом ошибочен или опасен и требуется возврат к предыдущему состоянию.

Откат выполняет только Пользователь. Ассистент рекомендует объём отката и объясняет причину. Агент не выполняет Git-откат.

## Единый краткий формат отчёта

Других обязательных или альтернативных шаблонов ОТЧЁТА АГЕНТА в постоянных документах быть не должно.

>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
НАЧАЛО ОТЧЁТА
>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>

ОТЧЁТ АГЕНТА

TASK: TASK XXX

Режим: Ограниченный / Самостоятельный

Степень выполнения:

- выполнено полностью;
- выполнено частично;
- не выполнено.

Изменено файлов: N

Кратко выполнено:

- действие;
- действие.

Переименованные, перемещённые или удалённые файлы:

- отсутствуют;

или:

- краткий список.

Резервная копия:

- путь;
- результат проверки SHA-256.

Обнаруженные проблемы:

- отсутствуют;

или:

- краткий список.

Git-команды: не выполнялись.

Commit: нет.

Push: нет.

Ожидает Commit/Push Пользователя и проверки Ассистента.

>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>
КОНЕЦ ОТЧЁТА
>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>>

После конца отчёта Агент выводит ровно одну строку:

- `Задача выполнена.`;
- `Задача частично выполнена.`;
- `Задача не выполнена.`.

Отчёт должен быть кратким и не дублировать полное содержание TASK или изменённых документов. Допускается только важное предупреждение, без которого Пользователь может принять неправильное решение.

## Изменение постоянных правил

Постоянные правила изменяются только по отдельной TASK, основанной на архитектурном решении Пользователя. История изменений хранится в Git; отдельный документ истории не создаётся.
