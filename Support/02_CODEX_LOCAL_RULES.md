# 02. CODEX LOCAL RULES — правила локальной работы Codex

## Назначение документа

Этот документ описывает ограничения и правила для Codex при работе в локальном репозитории `OziWal`.

Codex должен выполнять только текущую задачу пользователя и не делать лишних изменений.

## Обязательные документы перед работой

Перед любой задачей по OZON Codex должен учитывать:

1. `Support/00_BIG_TASK_CONTEXT.md`
2. `Support/01_START_HERE_CARD_RULES.md`
3. `Support/02_CODEX_LOCAL_RULES.md`

Если задача связана с Excel, дополнительно учитывать:

4. `Support/06_EXCEL_SAFETY_RULES.md`

Текущая задача хранится в `БЛОКЕ 1` файла `Support/00_BIG_TASK_CONTEXT.md`.

## Главные правила Codex

Codex обязан:

1. Выполнять только текущую задачу.
2. Не менять архитектуру проекта без отдельной задачи.
3. Не удалять файлы без отдельного разрешения.
4. Не переименовывать файлы без отдельного разрешения.
5. Не переносить файлы без отдельного разрешения.
6. Не создавать лишние файлы.
7. Не менять Excel-файлы без прямого указания.
8. Не принимать маркетинговые решения самостоятельно.
9. Не выдумывать характеристики товара.
10. Все спорные поля помечать как `ТРЕБУЕТ УТОЧНЕНИЯ`.
11. Если задача аналитическая — ничего не изменять.
12. Если задача требует правки — менять только явно указанные файлы.
13. После выполнения задачи возвращать отчёт.

## Ограничения по Excel

Codex не имеет права:

- менять структуру Excel-файлов без отдельного разрешения;
- удалять листы;
- переименовывать листы;
- менять порядок колонок;
- удалять формулы;
- перезаписывать формулы значениями;
- пересохранять Excel-файл просто ради форматирования.

Подробные правила находятся в:

`Support/06_EXCEL_SAFETY_RULES.md`

## Правило для task-loop документа

Файл:

`Support/00_BIG_TASK_CONTEXT.md`

является рабочим shell-документом агентского цикла OZON.

Codex выполняет только задачу из `БЛОКА 1 — ТЕКУЩАЯ ЗАДАЧА ДЛЯ CODEX`.

Большой task-документ не является постоянным правилом целиком.

Постоянные правила должны храниться в отдельных документах `Support/`.

Если задача явно разрешает, Codex может обновить `БЛОК 2 — ПОСЛЕДНИЙ CODEX REPORT` и `БЛОК 3 — REVIEW / STATUS`.

Если задача не разрешает, Codex не должен менять отчётные блоки.

## Правило формата задач от ассистента

Все задачи, которые ассистент готовит для вставки в чат Codex, должны выдаваться одним цельным Markdown-блоком.

Ассистент не должен дробить задачу на несколько отдельных блоков кода, если пользователь просит задачу для Codex.

Требования к задаче для Codex:

1. Вся задача должна быть внутри одного Markdown code block.
2. Внутри задачи должны быть чёткие блоки:
   - текущая задача;
   - что прочитать перед началом;
   - контекст;
   - разрешённые файлы;
   - запрещённые действия;
   - критерии готовности;
   - ожидаемый отчёт.
3. Если задача должна быть внесена в `Support/00_BIG_TASK_CONTEXT.md`, это нужно явно указать.
4. Если задача отправляется напрямую в чат Codex, это нужно явно указать.
5. Пути к документации проекта должны использовать актуальную папку `Support/`.
6. Если пользователь просит "задачу для Codex", по умолчанию готовить её одним блоком.

## Формат отчёта

После выполнения задачи Codex должен вернуть отчёт:

### CODEX REPORT

#### DONE

Что сделано.

#### CHANGED FILES

Какие файлы созданы или изменены.

#### NOTES

Важные замечания.

#### QUESTIONS

Вопросы к пользователю, если они есть.
