# Правила запуска задач Агента

> Навигация: [[Support/Карта_документации|Карта документации]] · Связано: [[Support/Правила_агентского_цикла|Правила агентского цикла]]

Этот документ фиксирует короткие правила запуска задач и завершения работы в документе цикла задач проекта.

## Правило подачи задач от ассистента

Если Пользователь просит подготовить задачу для Агента, Ассистент по умолчанию выдаёт два отдельных блока:

1. Большая задача.
2. Короткое сообщение для чата Агента.

## Большая задача

Большая задача предназначена для вставки в рабочий task-документ проекта, например:

`Support/00_BIG_TASK_CONTEXT.md`

Большая задача должна быть подробной и содержать:

- текущую задачу;
- что прочитать перед началом;
- контекст;
- разрешённые файлы;
- запрещённые действия;
- ограничения;
- критерии готовности;
- ожидаемый отчёт.

Большая задача должна быть выдана одним цельным Markdown-блоком.

Внутри большой задачи нельзя использовать вложенные fenced code blocks, потому что они создают отдельные кнопки копирования и визуально разваливают задачу.

Если внутри большой задачи нужно показать пример, использовать обычный текст, списки, цитату или отступы, но не отдельный fenced code block.

## Короткое сообщение для чата Агента

Короткое сообщение предназначено для запуска задачи в чате Агента.

Оно должно быть отдельным блоком после большой задачи.

Короткое сообщение должно быть коротким и понятным.

Оно обычно содержит:

- какой task-документ прочитать;
- какой блок выполнить;
- какие основные правила соблюдать;
- что вернуть после выполнения.

Пример смысла короткого сообщения:

    Прочитай `Support/00_BIG_TASK_CONTEXT.md`.
    Выполни только задачу из `БЛОКА 1 — ТЕКУЩАЯ ЗАДАЧА ДЛЯ АГЕНТА`.
    Соблюдай `Support/Правила_запуска_задач.md` и `Support/02_ПРАВИЛА_ЛОКАЛЬНОЙ_РАБОТЫ_АГЕНТА.md`.
    Не делай Commit и Push.
    После выполнения верни ОТЧЁТ АГЕНТА.

## Главное правило

Большая задача и короткое сообщение — это два разных блока.

Ассистент не должен забывать короткое сообщение для чата Агента, если Пользователь просит подготовить задачу для Агента.

Если пользователь явно просит только большую задачу или только короткое сообщение, ассистент выполняет именно это.

## Правила Git / репозитория

- Пользователь обычно сам делает Commit и Push.
- Агент не делает Commit и Push без прямого указания.
- После отчёта Агента Ассистент проверяет GitHub.
- Ассистент сверяет последний Commit с предыдущим известным состоянием.
- Если новых изменений нет, Ассистент сообщает, что Push не обнаружен.
- Ассистент не утверждает, что видит локальные изменения.
- В начале проверки Ассистент выводит `СТАТУС КОММИТА`.

После отчёта Агента Ассистент обязан попытаться получить именно сообщение последнего Commit на актуальной ветке.

Формат начала проверки:

    СТАТУС КОММИТА: "<сообщение Commit>"
    Repository: owner/repo
    Branch: main
    Commit: <short sha>
    Проверка: изменения найдены / Push не обнаружен / требуется уточнение

Если сообщение Commit доступно, Ассистент указывает именно его.

Если сообщение Commit короткое или неинформативное, например `+`, `01`, `02`, Ассистент всё равно указывает его как есть.

Пример:

    СТАТУС КОММИТА: "01"
    Commit: 94128b6

Если таких Commit несколько или сообщение не помогает понять, какой Commit проверяется, Ассистент дополнительно указывает короткий SHA.

Если сообщение Commit получить не удалось, Ассистент честно пишет, что сообщение не подтверждено, и указывает доступную ссылку Git, короткий SHA или результат сравнения.

## Правило завершения задачи в документе цикла задач

Если задача выполняется через рабочий документ цикла задач, например:

`Support/00_BIG_TASK_CONTEXT.md`

то после успешного выполнения задачи Агент должен:

1. Перенести отчёт о выполнении в `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА`.
2. Внести короткое резюме выполненной задачи в `БЛОК 3 — ПРОВЕРКА / СТАТУС`.
3. Очистить `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА ДЛЯ АГЕНТА`.
4. Вместо старой задачи оставить короткую заглушку:

    БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА

    Активной задачи нет.

    Статус:
    Ожидается новая задача от пользователя.

Важно:

- Агент не должен оставлять выполненную большую задачу в `БЛОКЕ 1`.
- Агент не должен удалять весь `БЛОК 1` целиком.
- Агент должен оставить структуру блока и маркеры `НАЧАЛО ЗАДАЧИ` / `КОНЕЦ ЗАДАЧИ`.
- Если задача выполнена не полностью, заблокирована или есть вопросы, Агент не должен полностью очищать `БЛОК 1`.
- Если задача выполнена не полностью, в `БЛОКЕ 1` нужно кратко указать статус и вопросы.
- Полную проверку выполняет Ассистент после проверки репозитория.
- Агент может записать в `БЛОК 3` только предварительный статус, например: `Ожидает проверки Ассистентом`.
