# Правила запуска задач Агента

> Навигация: [[Support/Карта_документации|Карта документации]]

## Назначение

Этот документ является единым источником истины для публикации TASK и обновления БЛОКОВ 1, 2 и 3.

Роли, режимы, Git-политика, жизненный цикл, формат ОТЧЁТА АГЕНТА и REVIEW определены только в `Support/Правила_агентского_цикла.md`.

## Два блока публикации

По умолчанию Ассистент готовит два отдельных копируемых блока:

1. полную TASK;
2. короткое сообщение запуска для чата Агента.

Если Пользователь явно просит только один блок, Ассистент готовит только его.

## Полная TASK

Полная TASK предназначена для БЛОКА 1 файла `Support/00_BIG_TASK_CONTEXT.md`.

Она должна содержать:

- номер и название TASK;
- режим выполнения;
- рабочую среду и локальный путь;
- цель и контекст;
- документы для чтения;
- разрешённые файлы и действия;
- запрещённые действия;
- требования к резервной копии;
- критерии готовности;
- требования к локальной проверке;
- требования к обновлению БЛОКОВ 1, 2 и 3.

Полная TASK публикуется одним цельным Markdown-блоком. Вложенные блоки кода с тройными обратными кавычками не используются без необходимости.

## Короткое сообщение запуска

Короткое сообщение предназначено для чата Агента и публикуется отдельно от полной TASK.

Оно кратко указывает:

- где находится активная TASK;
- какой режим использовать;
- ключевые ограничения;
- необходимость резервной копии;
- требование вернуть ОТЧЁТ АГЕНТА.

Пример смысла:

    Прочитай БЛОК 1 файла `Support/00_BIG_TASK_CONTEXT.md`.
    Выполни активную TASK в указанном режиме.
    Соблюдай постоянные правила проекта.
    Создай и проверь резервную копию.
    После выполнения обнови БЛОКИ 1, 2 и 3 и верни ОТЧЁТ АГЕНТА.

## Публикация в БЛОКЕ 1

Пользователь помещает полную TASK между маркерами `НАЧАЛО ЗАДАЧИ` и `КОНЕЦ ЗАДАЧИ`.

После публикации требования и критерии TASK не переписываются задним числом. Подробное правило неизменяемости находится в `Support/Правила_агентского_цикла.md`.

## Обновление БЛОКОВ после выполнения

При полном выполнении Агент:

1. записывает краткий фактический отчёт в БЛОК 2;
2. заменяет содержимое БЛОКА 3 одним последним предварительным статусом;
3. очищает содержимое БЛОКА 1, сохраняя заголовки и маркеры;
4. оставляет в БЛОКЕ 1:

    Активная задача отсутствует.

    Статус:
    Ожидается новая задача от Пользователя.

В БЛОКЕ 3 старые статусы не накапливаются.

При частичном выполнении, ошибке или блокировке Агент:

- не очищает БЛОК 1 полностью;
- кратко фиксирует текущий статус и препятствие;
- записывает фактический отчёт и не объявляет TASK принятой.

Окончательное решение после Commit и Push принимает Ассистент по правилам `Support/Правила_агентского_цикла.md`.
