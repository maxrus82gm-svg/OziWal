# 01. Правила составления карточек товаров OZON

> Навигация: [[Support/Карта_документации|Карта документации]]

## Назначение документа

Этот документ является стартовой точкой для любой работы по карточкам товаров OZON в проекте `OziWal`.

Перед созданием, изменением, проверкой или подготовкой карточки товара нужно сначала учитывать этот файл.

## Главная цель

Цель проекта — выработать повторяемый процесс подготовки карточек товаров для OZON на основе:

- фотографии товара;
- названия модели;
- дополнительных данных от пользователя;
- правил заполнения карточки;
- шаблонов задач для Ассистента и Агента;
- будущих Excel, CSV, JSON или API-форматов.

## Базовый принцип

Нельзя выдумывать точные характеристики товара, которые невозможно достоверно определить по фотографии или входным данным пользователя.

Если данных не хватает, поле нужно помечать как:

`ТРЕБУЕТ УТОЧНЕНИЯ`

## Что можно определять по фотографии

По фотографии можно осторожно описывать:

- форму товара;
- цвет;
- внешний вид;
- стиль;
- декоративные особенности;
- видимые элементы конструкции;
- примерное назначение, если оно очевидно;
- визуальное впечатление от товара;
- наличие отверстий, креплений, рельефа, фактуры, если это видно.

## Что нельзя точно утверждать по фотографии

По фотографии нельзя точно утверждать:

- материал;
- точный размер;
- вес;
- прочность;
- срок службы;
- страну производства;
- гарантию;
- комплектацию, если она не показана явно;
- безопасность для детей;
- безопасность для еды;
- безопасность для животных;
- совместимость с конкретными устройствами;
- наличие сертификатов;
- заводское качество поверхности;
- точную технологию изготовления, если пользователь её не подтвердил.

## Правило источников данных

Каждое спорное значение должно иметь понятный источник.

Использовать такие источники:

| Источник | Значение |
|---|---|
| `пользователь` | значение прямо указал пользователь |
| `фото` | значение видно на изображении |
| `вывод по фото` | осторожный вывод по внешнему виду |
| `шаблон` | значение взято из принятого шаблона проекта |
| `требует уточнения` | данных недостаточно |

## Правило для 3D-печатных товаров

Для 3D-печатных товаров нельзя автоматически обещать идеальную заводскую поверхность.

Допустимые формулировки:

- `изделие изготовлено методом 3D-печати`;
- `возможны небольшие следы послойной печати`;
- `каждое изделие может немного отличаться по оттенку и фактуре`;
- `подходит как декоративный аксессуар`, если это соответствует товару.

## Запрещённые или опасные формулировки

Не использовать без подтверждения:

- `лучший на рынке`;
- `вечный`;
- `неубиваемый`;
- `гарантированно`;
- `абсолютно безопасный`;
- `экологичный`;
- `сертифицированный`;
- `оригинальный брендовый товар`;
- `подходит для детей`, если это не подтверждено;
- `пищевой пластик`, если это не подтверждено.

## Минимальные данные от пользователя

Для нормальной карточки желательно получить:

- фото товара;
- название модели;
- категорию или предполагаемую категорию;
- материал;
- размер;
- цвет;
- комплектацию;
- цену;
- артикул или правило генерации артикула;
- остатки или способ производства под заказ;
- комментарии по назначению товара.

Если пользователь дал только фото и название, можно подготовить черновик карточки, но обязательно отметить недостающие поля.

## Рабочий маршрут

1. Пользователь передаёт фото и название модели.
2. Ассистент анализирует видимые признаки.
3. Создаётся черновик карточки.
4. Данные делятся на подтверждённые и требующие уточнения.
5. Пользователь проверяет спорные данные.
6. После проверки карточка может быть перенесена в Excel, CSV, JSON или API-формат.
7. Агент работает только с теми файлами и полями, которые указаны в задаче.

## Что считать ошибкой

Ошибкой считается:

- выдумывание точных характеристик;
- смешивание подтверждённых данных и предположений;
- отсутствие списка полей, которые требуют уточнения;
- изменение Excel-файлов без разрешения;
- перезапись шаблонов без задачи;
- удаление или переименование файлов без явного указания;
- использование рекламных обещаний без подтверждения.
