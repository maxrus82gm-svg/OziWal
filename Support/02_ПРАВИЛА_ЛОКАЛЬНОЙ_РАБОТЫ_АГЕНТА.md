# 02. Правила локальной работы Агента

> Навигация: [[Support/Карта_документации|Карта документации]] · Связано: [[Support/Правила_агентского_цикла|Правила агентского цикла]]

## Назначение документа

Этот документ описывает ограничения и правила для Агента при работе в локальном репозитории `OziWal`.

Агент должен выполнять только текущую задачу пользователя и не делать лишних изменений.

## Обязательные документы перед работой

Перед любой задачей по OZON Агент должен учитывать:

1. `Support/00_BIG_TASK_CONTEXT.md`
2. `Support/01_ПРАВИЛА_КАРТОЧЕК_ТОВАРОВ.md`
3. `Support/02_ПРАВИЛА_ЛОКАЛЬНОЙ_РАБОТЫ_АГЕНТА.md`

Если задача связана с Excel, дополнительно учитывать:

4. `Support/06_ПРАВИЛА_БЕЗОПАСНОЙ_РАБОТЫ_С_EXCEL.md`

Текущая задача хранится в `БЛОКЕ 1` файла `Support/00_BIG_TASK_CONTEXT.md`.

## Главные правила Агента

Агент обязан:

1. Выполнять только текущую задачу.
2. Не менять архитектуру проекта без отдельной задачи.
3. Не удалять файлы без отдельного разрешения.
4. Не переименовывать файлы без отдельного разрешения.
5. Не переносить файлы без отдельного разрешения.
6. Не создавать лишние файлы.
7. Не менять Excel-файлы без прямого указания.
8. Не принимать маркетинговые решения самостоятельно.
9. Не выдумывать характеристики товара.
10. Все спорные поля помечать как `ТРЕБУЕТ УТОЧНЕНИЯ`.
11. Если задача аналитическая — ничего не изменять.
12. Если задача требует правки — менять только явно указанные файлы.
13. После выполнения задачи возвращать отчёт.

## Ограничения по Excel

Агент не имеет права:

- менять структуру Excel-файлов без отдельного разрешения;
- удалять листы;
- переименовывать листы;
- менять порядок колонок;
- удалять формулы;
- перезаписывать формулы значениями;
- пересохранять Excel-файл просто ради форматирования.

Подробные правила находятся в:

`Support/06_ПРАВИЛА_БЕЗОПАСНОЙ_РАБОТЫ_С_EXCEL.md`

## Правило для рабочего документа цикла задач

Файл:

`Support/00_BIG_TASK_CONTEXT.md`

является рабочим документом агентского цикла OZON.

Агент выполняет только задачу из `БЛОКА 1 — ТЕКУЩАЯ ЗАДАЧА ДЛЯ АГЕНТА`.

Большой документ задачи не является постоянным правилом целиком.

Постоянные правила должны храниться в отдельных документах `Support/`.

Если задача явно разрешает, Агент может обновить `БЛОК 2 — ПОСЛЕДНИЙ ОТЧЁТ АГЕНТА` и `БЛОК 3 — ПРОВЕРКА / СТАТУС`.

Если задача не разрешает, Агент не должен менять отчётные блоки.

## Правило формата задач от ассистента

Подробные правила подачи задач и завершения цикла задач описаны в `Support/Правила_запуска_задач.md`.

Кратко:

1. Задачи от Ассистента для Агента должны выдаваться двумя отдельными блоками:
   - большая задача;
   - короткое сообщение для чата Codex.
2. Большая задача должна быть одним цельным Markdown-блоком.
3. Внутри большой задачи запрещены вложенные fenced code blocks.
4. После успешного выполнения задачи Агент очищает `БЛОК 1` рабочего документа цикла задач.
5. Отчёт переносится в `БЛОК 2`.
6. Краткий статус переносится в `БЛОК 3`.
7. Полную проверку выполняет Ассистент после проверки репозитория.

## Правило Git / репозитория

Полные правила Git и статуса Commit описаны в `Support/Правила_запуска_задач.md`.

Кратко:

- пользователь обычно сам делает commit и push;
- Агент не делает Commit и Push без прямого указания;
- после отчёта Агента Ассистент проверяет актуальный Commit в GitHub;
- Ассистент по возможности выводит сообщение последнего Commit как `СТАТУС КОММИТА`.

## Формат отчёта

После выполнения задачи Агент должен вернуть отчёт:

### ОТЧЁТ АГЕНТА

#### Выполнено

Что сделано.

#### Изменённые файлы

Какие файлы созданы или изменены.

#### Примечания

Важные замечания.

#### Вопросы

Вопросы к пользователю, если они есть.
