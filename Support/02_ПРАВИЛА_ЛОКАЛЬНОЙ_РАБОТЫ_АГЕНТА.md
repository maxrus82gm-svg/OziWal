# 02. Правила локальной работы Агента

> Навигация: [[Support/Карта_документации|Карта документации]] · Связано: [[Support/Правила_агентского_цикла|Правила агентского цикла]] · [[Support/Правила_запуска_задач|Правила запуска задач]]

## Назначение

Этот документ является единым источником истины для безопасной локальной работы Агента с файлами проекта `OziWal`.

Роли, режимы, жизненный цикл, формат отчёта и REVIEW определены в [[Support/Правила_агентского_цикла|Правилах агентского цикла]]. Публикация TASK и обновление БЛОКОВ 1–3 определены в [[Support/Правила_запуска_задач|Правилах запуска задач]].

## Границы локальной работы

Агент обязан:

- работать только с локальными файлами;
- выполнять только активную TASK;
- изменять только явно разрешённые файлы;
- не расширять архитектуру и границы TASK;
- не удалять, не переименовывать и не перемещать файлы без разрешения;
- не создавать лишние файлы;
- сохранять существующие изменения Пользователя;
- прекращать работу при ошибке, способной повредить документ.

Любые Git-команды запрещены согласно [[Support/Правила_агентского_цикла|Правилам агентского цикла]].

## Резервные копии

Если TASK требует резервного копирования, Агент до первого изменения:

1. определяет полный список изменяемых файлов;
2. проверяет существование и ненулевой размер каждого файла;
3. создаёт копии с сохранением понятной структуры путей;
4. хранит копии только вне репозитория и Obsidian-хранилища;
5. не использует `%TEMP%` для резервных копий;
6. не перезаписывает существующую копию и использует последовательный суффикс `_02`, `_03` и далее;
7. сверяет SHA-256 исходника и копии.

Если копирование или проверка SHA-256 не выполнены, изменение файлов запрещено.

Резервные копии не восстанавливаются автоматически. Восстановление выполняется только по отдельному решению Пользователя.

## Windows и PowerShell

Для локальной работы используются средства Windows и PowerShell.

Правила:

- использовать полные названия cmdlet;
- для путей использовать `-LiteralPath`, если cmdlet поддерживает этот параметр;
- перед удалением или рекурсивным перемещением проверять полный абсолютный путь;
- не выполнять рекурсивные операции по вычисленному или непроверенному пути;
- не передавать списки путей между разными оболочками для удаления или перемещения;
- использовать одну оболочку от проверки до операции.

Безопасные cmdlet чтения, копирования и проверки:

- `Get-Content`;
- `Test-Path`;
- `Copy-Item`;
- `Get-FileHash`;
- `Get-Item`;
- `Get-ChildItem`.

Без отдельной проверенной необходимости не использовать Unix-команды:

- `cat`;
- `head`;
- `tail`;
- `grep`;
- `sed`;
- `awk`.

## UTF-8 и кодировка

При чтении и записи Markdown необходимо явно использовать UTF-8.

Если текст в терминале выглядит нечитаемым:

1. не считать это доказательством повреждения файла;
2. повторно прочитать файл с явным указанием UTF-8;
3. не перезаписывать файл при подозрении на проблему кодировки;
4. сравнить размер и при наличии копии SHA-256;
5. если корректное чтение не удалось, остановиться и сообщить о проблеме.

Нельзя исправлять предполагаемую кодировку массовой перезаписью непроверенного содержимого.

## Безопасное изменение документов

Перед правкой Агент полностью читает затрагиваемый раздел и проверяет:

- назначение документа;
- разрешённый объём изменений;
- существующие ссылки и упоминания пути;
- необходимость локальной, а не полной перезаписи;
- отсутствие параллельного изменения файла внешней программой.

После правки Агент:

- полностью перечитывает изменённый документ с UTF-8;
- проверяет ненулевой размер;
- проверяет требуемые заголовки, маркеры и ссылки;
- убеждается, что не изменены запрещённые файлы;
- честно фиксирует ошибки и неполное выполнение.

Случайная, непроверенная или форматирующая перезапись документа запрещена.

## Рабочий документ TASK

Активная TASK находится в БЛОКЕ 1 файла `Support/00_BIG_TASK_CONTEXT.md`.

Агент работает только с активной TASK. Правила публикации, очистки БЛОКА 1 и обновления БЛОКОВ 2–3 находятся в [[Support/Правила_запуска_задач|Правилах запуска задач]].
