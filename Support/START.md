# START — базовые правила запуска задач Codex

Этот документ фиксирует короткие правила запуска задач и завершения работы в task-loop документе проекта.

## Правило подачи задач от ассистента

Если пользователь просит подготовить задачу для Codex, ассистент по умолчанию выдаёт два отдельных блока:

1. Большая задача.
2. Короткое сообщение для чата Codex.

## Большая задача

Большая задача предназначена для вставки в рабочий task-документ проекта, например:

`Support/00_BIG_TASK_CONTEXT.md`

Большая задача должна быть подробной и содержать:

- текущую задачу;
- что прочитать перед началом;
- контекст;
- разрешённые файлы;
- запрещённые действия;
- ограничения;
- критерии готовности;
- ожидаемый отчёт.

Большая задача должна быть выдана одним цельным Markdown-блоком.

Внутри большой задачи нельзя использовать вложенные fenced code blocks, потому что они создают отдельные кнопки копирования и визуально разваливают задачу.

Если внутри большой задачи нужно показать пример, использовать обычный текст, списки, цитату или отступы, но не отдельный fenced code block.

## Короткое сообщение для чата Codex

Короткое сообщение предназначено для запуска задачи в чате Codex.

Оно должно быть отдельным блоком после большой задачи.

Короткое сообщение должно быть коротким и понятным.

Оно обычно содержит:

- какой task-документ прочитать;
- какой блок выполнить;
- какие основные правила соблюдать;
- что вернуть после выполнения.

Пример смысла короткого сообщения:

    Прочитай `Support/00_BIG_TASK_CONTEXT.md`.
    Выполни только задачу из `БЛОКА 1 — ТЕКУЩАЯ ЗАДАЧА ДЛЯ CODEX`.
    Соблюдай `Support/START.md` и `Support/02_CODEX_LOCAL_RULES.md`.
    Не делай commit и push.
    После выполнения верни CODEX REPORT.

## Главное правило

Большая задача и короткое сообщение — это два разных блока.

Ассистент не должен забывать короткое сообщение для чата Codex, если пользователь просит подготовить задачу для агента.

Если пользователь явно просит только большую задачу или только короткое сообщение, ассистент выполняет именно это.

## Правило завершения задачи в task-loop документе

Если задача выполняется через рабочий task-loop документ, например:

`Support/00_BIG_TASK_CONTEXT.md`

то после успешного выполнения задачи Codex должен:

1. Перенести отчёт о выполнении в `БЛОК 2 — ПОСЛЕДНИЙ CODEX REPORT`.
2. Внести короткое резюме выполненной задачи в `БЛОК 3 — REVIEW / STATUS`.
3. Очистить `БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА ДЛЯ CODEX`.
4. Вместо старой задачи оставить короткую заглушку:

    БЛОК 1 — ТЕКУЩАЯ ЗАДАЧА

    Активной задачи нет.

    Статус:
    Ожидается новая задача от пользователя.

Важно:

- Codex не должен оставлять выполненную большую задачу в `БЛОКЕ 1`.
- Codex не должен удалять весь `БЛОК 1` целиком.
- Codex должен оставить структуру блока и маркеры `НАЧАЛО ЗАДАЧИ` / `КОНЕЦ ЗАДАЧИ`.
- Если задача выполнена не полностью, заблокирована или есть вопросы, Codex не должен полностью очищать `БЛОК 1`.
- Если задача выполнена не полностью, в `БЛОКЕ 1` нужно кратко указать статус и вопросы.
- Полный REVIEW делает ассистент после проверки репозитория.
- Codex может записать в `БЛОК 3` только предварительный статус, например: `Ожидает проверки ассистентом`.
