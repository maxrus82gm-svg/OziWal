## БЛОК 3. КОНТЕКСТ

Пользователь является 3D-моделлером и хочет выбирать, какие модели стоит делать и печатать для продажи на маркетплейсах.

Главная бизнес-цель:

Понять, какие 3D-печатные товары имеют спрос на OZON / Wildberries, до того как пользователь потратит время на моделирование, печать, фото и карточку товара.

Будущий marketparser нужен не просто для скачивания данных, а для оценки товарных идей.

Нужно будет проверять:

- есть ли спрос;
- насколько сильная конкуренция;
- какие цены в выдаче;
- сколько отзывов у похожих товаров;
- какие названия используют конкуренты;
- можно ли сделать товар лучше;
- насколько товар подходит под 3D-печать;
- стоит ли пользователю моделировать такую идею.

## БЛОК 8. СОДЕРЖАНИЕ `marketparser/docs/01_DEMAND_RESEARCH_WORKFLOW.md`

Создать файл:

`marketparser/docs/01_DEMAND_RESEARCH_WORKFLOW.md`

Документ должен описывать процесс проверки идеи товара.

Процесс:

1. Пользователь или ассистент формирует список идей.
2. Для каждой идеи подбираются поисковые запросы.
3. Идеи проверяются на OZON / Wildberries.
4. Для каждого запроса собираются признаки спроса.
5. Для каждого запроса собираются признаки конкуренции.
6. Отдельно оценивается сложность моделирования и печати.
7. Идея получает итоговый приоритет.
8. Пользователь выбирает 3–5 лучших идей для моделирования и тестовой печати.

Описать, что такое признаки спроса:

- много отзывов у товаров в топе;
- есть товары с рейтингом;
- есть повторяющиеся карточки;
- есть похожие поисковые подсказки;
- есть понятная покупательская потребность.

Описать, что такое признаки конкуренции:

- очень много одинаковых товаров;
- низкие цены;
- сильные фото;
- много продавцов;
- много товаров с тысячами отзывов;
- сложно выделиться дизайном.

Описать, что такое признаки хорошей идеи для 3D-печати:

- небольшая модель;
- быстрый цикл печати;
- низкий риск брака;
- понятная польза;
- легко сфотографировать;
- можно сделать дизайн лучше конкурентов;
- есть возможность продавать набором;
- хорошая маржа.
