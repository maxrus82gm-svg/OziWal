# 00. BIG TASK CONTEXT — рабочий shell-документ OZON

Это рабочий shell-документ текущего агентского цикла по проекту OZON.

Он нужен для того, чтобы хранить:

- только активную задачу в `БЛОКЕ 1`;
- последний `CODEX REPORT` в `БЛОКЕ 2`;
- краткий `REVIEW / STATUS` в `БЛОКЕ 3`;
- короткую карту чтения документов `Support/`.

После успешного выполнения активная задача очищается из `БЛОКА 1`.
Старые выполненные задачи не должны накапливаться в `БЛОКЕ 1`.

Постоянные правила вынесены в отдельные документы:

- `Support/01_START_HERE_CARD_RULES.md`
- `Support/02_CODEX_LOCAL_RULES.md`
- `Support/03_WORKFLOW_SIMULATION.md`
- `Support/04_AGENT_TASK_TEMPLATE.md`
- `Support/05_PRODUCT_CARD_FIELDS.md`
- `Support/06_EXCEL_SAFETY_RULES.md`
- `Support/07_PRODUCT_CARD_EXAMPLE.md`

## Что читать всегда

Codex всегда читает:

1. `Support/00_BIG_TASK_CONTEXT.md`
2. `Support/01_START_HERE_CARD_RULES.md`
3. `Support/02_CODEX_LOCAL_RULES.md`

## Матрица чтения Support/

- `Новая карточка товара / описание по фото` -> `01_START_HERE_CARD_RULES.md` + `03_WORKFLOW_SIMULATION.md` + `04_AGENT_TASK_TEMPLATE.md` + `05_PRODUCT_CARD_FIELDS.md`
- `Работа Codex / ограничения / локальные изменения` -> `02_CODEX_LOCAL_RULES.md`
- `Excel / XLS / таблицы / шаблоны OZON` -> `06_EXCEL_SAFETY_RULES.md` + `05_PRODUCT_CARD_FIELDS.md`
- `Пример карточки / формат результата` -> `07_PRODUCT_CARD_EXAMPLE.md` + `04_AGENT_TASK_TEMPLATE.md`
- `Процесс от фото до карточки` -> `03_WORKFLOW_SIMULATION.md`
- `Поля карточки / характеристики / SEO` -> `05_PRODUCT_CARD_FIELDS.md`
- `История / статус / прошлые решения` -> `00_BIG_TASK_CONTEXT.md`
- `Парсер / исследование спроса / OZON / Wildberries / выбор товарной идеи` -> `marketparser/README.md` + `marketparser/docs/00_MARKETPARSER_RULES.md` + `marketparser/docs/01_DEMAND_RESEARCH_WORKFLOW.md` + `marketparser/docs/03_SAFE_PARSING_RULES.md` + `marketparser/docs/04_PRODUCT_IDEA_SCORING.md`

Если задача явно разрешает правку документации, Codex может обновлять:

- `БЛОК 2 — ПОСЛЕДНИЙ CODEX REPORT`
- `БЛОК 3 — REVIEW / STATUS`
- новые подтверждённые правила в нужном документе `Support/`

Без явного разрешения Codex не должен менять `БЛОК 2`, `БЛОК 3` и постоянные правила.
