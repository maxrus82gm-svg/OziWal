# 03. Процесс подготовки карточки товара

> Навигация: [[Support/Карта_документации|Карта документации]]

## Назначение документа

Этот документ описывает, как должен работать процесс подготовки карточки товара OZON.

Он нужен, чтобы до разработки Excel, CSV, JSON или API сначала проверить сам подход.

## Общая схема

Пользователь даёт:

- фотографию товара;
- название модели;
- дополнительные данные, если они есть.

Ассистент делает:

- анализ фото;
- черновик названия;
- короткое описание;
- полное описание;
- преимущества;
- SEO-ключи;
- список характеристик;
- список полей, которые требуют уточнения;
- задачу для Агента.

Агент делает:

- создаёт или обновляет документы;
- переносит данные в нужный файл;
- готовит структуру под Excel, CSV, JSON или API, если задача это разрешает;
- не выдумывает данные самостоятельно.

Пользователь делает:

- подтверждает материал;
- подтверждает размер;
- подтверждает цену;
- подтверждает комплектацию;
- проверяет спорные характеристики;
- утверждает карточку перед выгрузкой.

## Этап 1. Фото и название

На входе есть:

- фото товара;
- название модели.

На этом этапе можно подготовить только черновик.

Если не указаны материал, размер, вес, цена и комплектация, они должны быть помечены как `ТРЕБУЕТ УТОЧНЕНИЯ`.

## Этап 2. Анализ фото

Ассистент описывает только видимые признаки:

- форма;
- цвет;
- стиль;
- фактура;
- видимые элементы;
- возможное назначение.

Нельзя утверждать то, что не видно на фото и не указано пользователем.

## Этап 3. Черновик карточки

Черновик карточки должен содержать:

- название товара;
- короткое описание;
- полное описание;
- преимущества;
- характеристики;
- SEO-ключи;
- поля, которые требуют уточнения;
- комментарий агента.

## Этап 4. Проверка пользователем

Пользователь должен проверить:

- материал;
- размер;
- вес;
- цену;
- комплектацию;
- категорию;
- назначение;
- ограничения;
- необходимость указания 3D-печати.

## Этап 5. Перенос в рабочий формат

После проверки карточка может быть перенесена в:

- Markdown;
- Excel;
- CSV;
- JSON;
- API-формат OZON в будущем.

## Главный принцип

Сначала симулируем процесс в документах.

Только после того, как процесс понятен и стабилен, можно переходить к автоматизации через Excel или API.
